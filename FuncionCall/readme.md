
## Function Calling 特点总结

### 1. 核心定义
Function Calling（函数调用）是大语言模型（LLM）与外部工具/服务交互的关键机制，允许模型在需要时调用预定义的函数来获取实时数据或执行特定操作。

### 2. 主要特点

| 特点 | 说明 |
|------|------|
| **结构化输出** | 模型输出符合预定义JSON Schema的参数格式，便于程序解析 |
| **延迟执行** | 模型只生成调用请求，实际执行由外部系统完成 |
| **双向交互** | 支持"模型→系统→模型"的循环调用流程 |
| **动态决策** | 模型自主判断何时、调用哪个函数、传入什么参数 |

### 3. 主要缺点

1. 大模型依赖： 需要大模型支持 tools 的入参。
1. 绑定语言： 工具函数与Agent的工程是耦合在一起的，所以它们必须使用相同的语言实现。
2. 不够独立： 工具函数的实现依赖于Agent的工程，不能独立于Agent运行。
